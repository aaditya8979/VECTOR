The Challenge

Why small models fail
on real codebases

Three fundamental architectural problems that no prompt engineering can solve. VECTOR eliminates all three with deterministic engineering.

📏
Context Window Overflow
A 7B model sees 32K tokens. Flask alone contains 1,221 functions across dozens of files. The full codebase is 50× larger than the model's entire memory. You cannot stuff reality into a window.
# Flask codebase
1,221 functions → overflow
1,015 call edges → lost
32K limit → 50× too small
🎭
Hallucinated Symbols
Without precise context, language models confidently invent function names, object attributes, and API patterns that don't exist. The code looks valid. It crashes at runtime.
# Model generated (wrong):
ctx.request.start_time
AttributeError: 'Request'
has no 'start_time'
📍
Line-Number Blindness
Unified diff format requires exact byte positions. A 7B model cannot reliably count 1,600 lines. Wrong count corrupts the AST. We observed this failure five consecutive times before solving it with AST replacement.
# Model said:
@@ -1601,6 +1601,7 @@
# Reality: 20 lines, not 6
SyntaxError at line 1601
Core Innovation

From source code to
a living graph

VECTOR transforms your entire codebase into a Code Property Graph — a queryable, live-updating structural memory that lets the model understand relationships across thousands of files without seeing them.

SOURCE CODE
# flask/app.py
def full_dispatch_request(self, ctx: AppContext) → Response:
  """Dispatches the request and performs pre/post processing."""
  self._got_first_request = True
  try:
    rv = self.preprocess_request(ctx)
    if rv is None:
      rv = self.dispatch_request(ctx)
  except Exception as e:
    rv = self.handle_user_exception(ctx, e)
  return self.finalize_request(ctx, rv)
AST → CPG TRANSFORMATION
STEP 01 · PARSE
tree-sitter builds the Abstract Syntax Tree
Every function, class, and expression becomes a typed node. Byte-accurate positions. 8 languages with identical API. <50ms for any file size.
STEP 02 · LINK
Call graph edges connect the nodes across files
Who calls what. What imports what. Where data flows. 1,015 edges for Flask, built in 0.8 seconds. The AST becomes a Code Property Graph.
STEP 03 · COMPRESS
TSDC extracts the task-relevant subgraph
For one modification task: 1-hop callers, 1-hop callees, type contracts, codebase rules. 1M+ tokens compressed to ≤2,500. No LLM involved in compression. Pure symbolic extraction.
STEP 04 · MODIFY
7B model outputs the complete function. AST finds it and replaces.
No line numbers. No diff format. The model outputs clean code. tree-sitter finds the function boundaries exactly. The replacement is atomic and byte-perfect.
STEP 05 · LEARN
Every verified edit writes a Knowledge Item back into the graph
Rules extracted from verified code — not model output alone. After 50 tasks on a codebase, VECTOR knows patterns no external model does. It gets smarter with every commit.

Verification Engine

5 layers. No model.
No hallucinations escape.

Every verification layer is pure deterministic code. The model generates. The engine verifies. If any layer fails, a structured error is injected back into the TSDC document and the loop retries. Maximum 5 attempts. Original file never touched on failure.

1
Syntax Validation
ast.parse() for Python · tree-sitter ERROR node detection for TypeScript, Go, Rust, C++ · catches all syntax-level errors before any execution
INSTANT · <1ms

Symbol Hallucination Check
Every function called in the output must exist in TSDC Tier 3 (AVAILABLE CALLEES). Invented names like db.run() or ctx.request.start_time fail here immediately.
~10ms

Static Type Verification
Language-specific: mypy for Python, tsc for TypeScript, gofmt for Go, rustfmt for Rust. Wrong types, missing imports, incorrect return types all caught here.
~2s

Isolated subprocess import and smoke-call. Catches NameError, AttributeError, import crashes before the file is ever modified. The final safety net.

VECTOR: Verified Execution and Contextual Tree Orchestrator for Deterministic Repository-Level Code Modification with Small Language Models
Aaditya Agarwal · M4 MacBook Air · Offline
We present VECTOR, a deterministic code modification agent that enables a 7B language model to perform repository-level edits with accuracy comparable to 70B models — without fine-tuning, cloud APIs, or internet access. VECTOR introduces Task-Scoped Deterministic Context (TSDC), a zero-model symbolic compression method that reduces multi-million token codebases to ≤2,500 tokens per modification task via Code Property Graph traversal. A 5-layer verification pipeline ensures all generated code is syntactically valid, type-correct, and test-passing before any file is modified...
Code Generation LLM Agents Software Engineering On-device AI
The Core Claim
A 7B model operating on TSDC-compressed context (≤2,500 tokens) with a 5-layer deterministic verification loop matches or exceeds the code modification accuracy of models 10× its size on single-function tasks in tracked repositories — on consumer hardware in under 30 seconds.
🗜
Zero-model context compression
SWEzze needs a cross-encoder. LongCodeZip needs perplexity scoring. TSDC needs zero model calls — pure symbolic CPG traversal.
🌳
AST-based function replacement
Eliminates line-number hallucination entirely. Model outputs code. tree-sitter handles positioning. Byte-perfect, language-agnostic.
🧠
Session-persistent Knowledge Items
Seeded only from verified edits. Never from model output alone. The only anti-hallucination learning mechanism that cannot launder errors.
🌐
Multi-language unified pipeline
8 languages from one registry. Same CPG. Same verification. Same intelligence. The Language Registry itself is a publishable artifact.
